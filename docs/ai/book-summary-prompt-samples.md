# 책 요약 프롬프트 기준

> 관련 이슈: #102 `[feat/ai] 책 요약 생성 프롬프트 및 66권 데이터 생성`

## 목표

책 요약은 장 요약보다 길게 읽히는 안내 콘텐츠다. 사용자가 특정 책을 읽기 전에 전체 흐름, 주요 주제, 읽을 때 붙잡을 관점을 가볍게 파악하도록 돕는다.

장 선택 화면에서는 `shortSummary`만 짧게 보여 주고, 자세히 보기 화면이나 이후 AI 가이드 탭에서는 `summary`, `readingPoint`, `keywords`, `outline`을 함께 사용한다.

## 출력 필드

```json
{
  "bookCode": "창",
  "shortSummary": "장 선택 화면에 들어갈 1문장 요약",
  "summary": "상세 화면에 들어갈 2~4문장 설명",
  "readingPoint": "이 책을 읽을 때 붙잡을 관점 1문장",
  "keywords": ["핵심어1", "핵심어2", "핵심어3"],
  "outline": ["큰 흐름1", "큰 흐름2", "큰 흐름3"]
}
```

## 생성 원칙

- 특정 교파의 결론보다 성경 본문이 보여 주는 서사와 주제 흐름을 우선한다.
- 저자, 연대, 역사적 배경은 논쟁이 크면 단정하지 않는다.
- 설교체, 권면체, 묵상 적용문으로 쓰지 않는다.
- `shortSummary`는 1문장으로 짧게 쓴다.
- `summary`는 상세 화면에서 읽히는 문단이므로 2~4문장으로 자연스럽게 쓴다.
- `readingPoint`는 사용자가 책을 읽을 때 붙잡을 관점을 제시하되 명령형으로 쓰지 않는다.
- `keywords`는 4~6개, `outline`은 3~5개를 권장한다.

## System Prompt

```text
당신은 한국어 성경 앱에 들어갈 책 소개 콘텐츠를 만드는 보조자입니다.
성경 본문과 책의 큰 흐름을 존중하되, 특정 교파의 논쟁적 결론이나 과도한 해석을 앞세우지 않습니다.
결과는 앱 화면에서 바로 사용할 수 있도록 담백하고 정확한 한국어로 작성합니다.
반드시 순수 JSON 객체만 반환합니다.
```

## User Prompt Template

```text
다음 성경 책에 대한 앱용 책 요약 콘텐츠를 JSON으로 작성하세요.

입력:
- 책 코드: {bookCode}
- 책 이름: {bookName}
- 구약/신약: {testament}
- 장 수: {chapterCount}

작성 기준:
1. shortSummary: 장 선택 화면 상단에 들어갈 1문장 요약입니다.
2. summary: 상세 화면에 들어갈 2~4문장 설명입니다.
3. readingPoint: 이 책을 읽을 때 붙잡을 핵심 관점 1문장입니다.
4. keywords: 핵심 키워드 4~6개입니다.
5. outline: 책의 큰 흐름 3~5개입니다.
6. 설교, 권면, 묵상 적용, 과도한 교리 단정은 피합니다.
7. 논쟁적인 저자/연대/예언 해석은 단정하지 말고 본문 흐름 중심으로 씁니다.
8. 마크다운 코드 블록 없이 JSON 객체만 반환합니다.

출력:
{
  "bookCode": "...",
  "shortSummary": "...",
  "summary": "...",
  "readingPoint": "...",
  "keywords": ["...", "..."],
  "outline": ["...", "..."]
}
```

## 검수 기준

- 장 선택 화면에서 `shortSummary`만 읽어도 책의 방향이 잡히는가?
- 상세 화면에서 `summary`, `readingPoint`, `keywords`, `outline`이 서로 반복만 하지 않는가?
- 특정 교파의 논쟁적 표현이나 본문 밖 단정이 들어가지 않았는가?
- `keywords`와 `outline`이 FE에서 리스트로 보여 주기에 충분히 짧은가?
