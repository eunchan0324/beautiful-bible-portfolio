# Phase 12: 교회 일정 관리/알림 확장

> Phase 8의 정적 교회 일정 캘린더를 운영 가능한 구조로 확장한다.

---

## 배경

Phase 8에서는 운영 공수를 줄이기 위해 교회 일정을 정적 데이터로 관리한다.
실제로 교인들이 사용하고 일정 관리 필요가 확인되면, 관리자 기능과 알림을 붙인다.

---

## 목표

- 교회 일정 데이터를 백엔드에서 관리한다.
- 권한 있는 사용자만 일정을 추가/수정/삭제한다.
- PWA 알림 구조를 재사용해 일정 알림을 보낸다.

---

## MVP 확장 범위

포함한다.

- `CHURCH_ADMIN` 권한
- 일정 CRUD API
- 관리자용 일정 추가/수정/삭제 화면
- 일정 조회 API
- 일정 하루 전 또는 당일 알림

포함하지 않는다.

- 복잡한 조직/부서 권한
- 반복 일정 고급 규칙
- OS 기본 캘린더 자동 등록
- Google Calendar 양방향 동기화

---

## 데이터 초안

```text
church_events
- id
- title
- date
- time
- location
- description
- type
- created_by
- created_at
- updated_at
```

필요하면 나중에 반복 일정 테이블을 분리한다.

---

## 알림 구조

Phase 9의 PWA Web Push 구독 구조를 재사용한다.

```text
Scheduler
  -> 일정 알림 발송 API 호출
BE
  -> 알림 대상 일정 조회
  -> 구독자에게 Web Push 발송
```

---

## 운영 원칙

- 담당자가 생기기 전까지는 이 Phase를 진행하지 않는다.
- 일정 관리 화면은 최대한 단순하게 만든다.
- 잘못된 일정 알림이 나가는 리스크를 줄이기 위해 알림은 별도 확인 단계를 둘 수 있다.

---

## 완료 기준

- 관리자 권한 사용자가 일정을 관리할 수 있다.
- 일반 사용자는 일정 조회만 가능하다.
- 일정 알림을 테스트 발송할 수 있다.
- Phase 8의 정적 일정 데이터를 API 기반으로 대체할 수 있다.
