# Phase 10: 개인 통독 MVP 기획

> 이 문서는 #162 개인 통독 MVP의 제품 범위와 UX 정책을 정리한다.
> 단체 통독은 실제 수요와 운영 비용 전환 가능성이 확인된 뒤 별도 Phase로 확장한다.

## 배경

현재 Beautiful Bible은 포트폴리오 목적과 실제 교회 사용자 사용성을 함께 고려해 고도화하고 있다.
백엔드는 Render 무료 서버에서 운영 중이므로 cold start가 발생할 수 있고, 단체 통독처럼 그룹, 초대, 멤버별 진행률이 필요한 기능은 운영 부담이 커질 수 있다.

따라서 이번 단계에서는 단체 통독을 바로 구현하지 않고, 개인이 스스로 읽을 범위와 리듬을 정해 꾸준히 이어갈 수 있는 개인 통독 MVP에 집중한다.
단체 통독은 사용자가 "함께 하고 싶다"는 수요를 보이고 후원 또는 유료 서버 전환이 가능해졌을 때 확장한다.

---

## 제품 방향

개인 통독은 코스 카탈로그가 아니라, 사용자가 직접 읽을 책과 하루 목표를 정하는 기능으로 시작한다.
통독은 새로운 본문 뷰어가 아니라 기존 성경 읽기 화면 위에 "계획"과 "장 완료" 맥락을 얹는 기능이다.

핵심 경험은 다음과 같다.

```text
홈에서 나의 통독 확인
-> 통독 계획 만들기
-> 오늘 읽을 말씀 확인
-> 기존 성경 읽기 화면에서 본문 읽기
-> 장 맨 아래에서 읽기 완료
-> 진행률 확인
```

## 통독 계획 생성

사용자가 직접 정하는 값은 네 가지로 제한한다.

```text
1. 읽을 책
2. 하루 목표 장 수
3. 시작일
4. 통독 이름
```

### 읽을 책

책 단위로만 선택한다.

- 성경 전체
- 구약 전체
- 신약 전체
- 개별 책 여러 개

장 단위 선택은 MVP에서 제외한다.
장 단위 선택은 자유도는 높지만 시작 전 설정 피로도가 커지고, 일반적인 통독 앱에서도 핵심 흐름은 책 단위 선택에 가깝다.

선택된 책은 성경 기본 순서대로 진행한다.
드래그로 순서를 바꾸거나 같은 책을 여러 번 추가하는 기능은 제외한다.

### 책 선택 분류

책 선택 화면은 구약/신약과 성경 분류를 함께 보여준다.
분류 설명은 MVP에서 길게 제공하지 않고, 섹션 제목으로만 사용한다.

```text
구약
- 모세오경
- 역사서
- 시가서
- 대선지서
- 소선지서

신약
- 복음서
- 역사서
- 바울서신
- 공동서신
- 예언서
```

### 하루 목표 장 수

하루 목표는 직접 입력하게 한다.

```text
기본값: 1장
허용 범위: 1장 ~ 10장
```

하루 목표는 계획을 나누기 위한 기준값일 뿐, 사용자가 실제로 더 많이 읽는 것을 막지 않는다.
예를 들어 하루 목표가 2장이어도 사용자가 4장까지 읽고 완료하면 실제 진행률은 4장 기준으로 반영된다.

선택한 전체 분량이 하루 목표보다 적으면 남은 장까지만 배정한다.

```text
선택한 책: 오바댜
전체 분량: 1장
하루 목표: 10장
결과: 예상 기간 1일, 하루 1장
```

### 시작일

시작일 기본값은 오늘이다.
사용자가 내일이나 특정 날짜부터 시작하고 싶을 수 있으므로 날짜 변경은 허용한다.

종료일은 사용자가 직접 정하지 않고 앱이 자동 계산한다.

```text
요한복음 21장 / 하루 3장 / 오늘 시작
-> 예상 7일
```

시작일이 미래이면 홈과 통독 화면에서는 시작 예정 상태로 보여준다.
시작일이 지난 뒤 계획보다 덜 읽었더라도 밀렸다는 압박 문구는 사용하지 않는다.

### 통독 이름

통독 이름은 자동 생성값을 기본으로 제공하고 사용자가 수정할 수 있게 한다.
처음부터 빈칸으로 두면 작성 부담이 생기므로 자동 이름을 먼저 넣는다.

예시는 다음과 같다.

```text
요한복음 통독
창세기부터 출애굽기까지
선택한 3권 통독
성경 전체 통독
```

계획 설명 입력은 MVP에서 제외한다.
개인 통독의 핵심은 소개 글 작성이 아니라 읽기 시작이기 때문이다.
설명은 나중에 단체 통독으로 확장할 때 재검토한다.

## 홈 진입 정책

하단 네비게이션은 변경하지 않는다.

```text
홈 / 검색 / 성경 / 해설 / 마이
```

통독은 홈의 오늘의 말씀 아래에 배치한다.
홈은 "오늘의 말씀", "나의 통독", "우리 교회 일정"을 담는 오늘의 신앙생활 화면으로 확장한다.

홈 순서는 다음과 같다.

```text
1. 오늘의 말씀
2. 나의 통독
3. 우리 교회 일정
```

### 홈 카드 상태

로그인하지 않은 사용자는 다음을 보여준다.

```text
나의 통독
로그인하면 통독 계획을 만들고 진행률을 저장할 수 있어요.

[카카오로 로그인]
```

로그인했지만 진행 중 계획이 없으면 다음을 보여준다.

```text
나의 통독
읽을 책을 정하고 매일 이어가보세요.

[통독 계획 만들기]
```

진행 중 계획이 있으면 다음을 보여준다.

```text
나의 통독
요한복음 통독 · 38%

오늘 읽을 말씀
요한복음 5-6장

[이어 읽기]
```

완료한 통독은 홈에서 별도 카드로 보여주지 않는다.
선택한 모든 장을 완료하면 해당 계획은 완료 상태가 되고, 홈에서는 진행 중 계획 없음 상태로 취급한다.

## 라우트

MVP 라우트는 다음처럼 단순하게 시작한다.

```text
/reading
- 내 통독 화면 또는 빈 상태

/reading/new
- 통독 계획 만들기
```

진행 중 계획은 사용자당 1개만 허용하므로 `/reading/{planId}`는 MVP에서 필수로 만들지 않는다.
나중에 완료 기록 또는 여러 계획을 지원할 때 확장한다.

## 내 통독 화면

내 통독 화면은 진행 중인 통독 계획 하나를 보여주는 화면이다.
계획 생성 직후에는 이 화면 위에 오늘 읽을 말씀 바텀시트를 자동으로 연다.

계획 생성 직후 바텀시트 예시는 다음과 같다.

```text
오늘 읽을 말씀

DAY 1
창세기 1-3장

[바로 읽기 창세기 1장]
```

평소 `/reading` 진입 시에는 바텀시트를 자동으로 열지 않는다.
사용자가 오늘 읽을 말씀 카드를 누르면 열 수 있게 한다.

내 통독 화면은 다음 정보를 포함한다.

- 계획명
- 전체 진행률
- 오늘 읽을 말씀
- 이어 읽기 버튼
- 진도표
- 성경통독표
- 계획 삭제

진행 중 계획이 이미 있으면 새 계획을 바로 만들 수 없고 다음 안내를 보여준다.

```text
현재는 하나의 통독 계획만 진행할 수 있어요.
새 계획을 시작하려면 기존 계획을 완료하거나 삭제해주세요.
```

## 오늘 읽을 말씀

"오늘 읽을 말씀"은 날짜에 엄격하게 묶인 미션이 아니라, 아직 완료하지 않은 가장 앞 장부터 하루 목표 장 수만큼 보여주는 추천 분량이다.

예를 들어 하루 목표가 2장이고 요한복음 1장부터 시작한다면 다음처럼 보여준다.

```text
오늘 읽을 말씀
- 요한복음 1장
- 요한복음 2장
```

사용자가 요한복음 1~4장을 읽음 처리하면 다음에 보여줄 말씀은 요한복음 5장부터 시작한다.
이 방식은 날짜를 놓쳤을 때 사용자를 압박하기보다, 현재 위치에서 자연스럽게 이어 읽도록 돕는다.

## 진도표와 성경통독표

### 진도표

진도표는 하루 단위 흐름을 보여주는 표다.

```text
1일차  요한복음 1-4장  완료
2일차  요한복음 5-7장  오늘
3일차  요한복음 8-10장
4일차  요한복음 11-13장
```

사용자가 매일 확인해야 하는 것은 오늘 읽을 분량이므로 진도표를 성경통독표보다 위에 둔다.

### 성경통독표

성경통독표는 책별 장 완료 상태를 보여주는 그리드다.

```text
요한복음
[1] [2] [3] [4] [5] [6] ... [21]
```

완료한 장도 성경통독표에서 다시 열 수 있다.
다만 `이어 읽기`와 `바로 읽기`는 항상 아직 완료하지 않은 가장 앞 장으로 이동한다.

## 기존 성경 읽기 화면

본문 읽기는 기존 성경 읽기 화면을 재사용한다.

유지하는 기능은 다음과 같다.

- 구절 선택
- 하이라이트
- 저장
- 복사
- 공유
- 기존 하단 설정 UI

통독에서 읽는 동안에도 하이라이트와 저장은 동일한 `verse_key`를 사용한다.
따라서 통독 중 표시한 하이라이트는 일반 성경 읽기 화면에서도 그대로 보인다.

읽기 화면에는 퍼센트 기반 스크롤 진행률을 표시하지 않는다.
퍼센트는 사용자가 본문 자체보다 완료율에 집중하게 만들 수 있으므로, MVP에서는 제외한다.

대신 통독 계획에 포함된 장의 맨 아래에 통독 완료 CTA를 표시한다.
일반 성경 읽기 화면에서는 통독 완료 CTA가 보이지 않는다.

```text
요한복음 1장을 다 읽었어요.
[읽기 완료]
```

완료한 장을 다시 열면 다음처럼 표시한다.

```text
이미 읽었어요
```

완료 취소는 MVP에서 제외한다.

## 완료 피드백

읽기 완료 버튼을 누르면 장 완료를 저장하고 바텀시트를 보여준다.

다음 장이 있으면 다음처럼 보여준다.

```text
요한복음 1장을 읽었어요.

[확인] [다음 장 읽기]
```

오늘 읽을 말씀의 마지막 장이면 다음처럼 보여준다.

```text
오늘 읽을 말씀을 다 읽었어요.

[확인] [통독으로 돌아가기]
```

전체 계획의 마지막 장이면 다음처럼 보여준다.

```text
요한복음 통독을 완료했어요.

[확인] [통독 마치기]
```

`통독 마치기`를 누르면 계획 상태를 완료로 두고 홈에서는 진행 중 계획 없음 상태로 취급한다.
완료한 통독 목록, 완료 히스토리, 완료 통계는 MVP에서 제외한다.

---

## 완료 정책

읽음 완료는 장 단위로 저장한다.
구절 단위 스크롤 위치는 저장하지 않는다.

장 맨 아래의 `읽기 완료` 버튼을 눌러야 완료된다.
스크롤만 끝까지 내렸다고 자동 완료하지 않는다.

선택한 모든 장이 완료되면 계획은 자동으로 완료 가능 상태가 된다.
완료 후에는 새 통독 계획을 만들 수 있다.

---

## 삭제 정책

MVP에서는 계획 삭제만 제공한다.
계획 수정은 제외한다.

삭제 확인 문구는 읽음 기록이 함께 삭제된다는 점을 분명히 알려야 한다.

```text
통독 계획을 삭제할까요?
읽음 기록도 함께 삭제돼요.
```

## 데이터 설계 방향

초안은 다음 구조를 기준으로 한다.

```text
reading_plans
- id
- user_id
- title
- start_date
- daily_chapter_target
- status
- created_at

reading_plan_books
- id
- plan_id
- book_code
- book_order

reading_plan_items
- id
- plan_id
- day_number
- book_code
- chapter_num
- item_order

reading_progress
- id
- plan_id
- user_id
- book_code
- chapter_num
- completed_at
```

향후 단체 통독으로 확장할 때는 `reading_progress`의 장 단위 완료 데이터를 재사용하고, 그룹/멤버십/초대/그룹 계획을 별도 도메인으로 추가한다.

## 무료 서버 전략

Render 무료 서버의 cold start를 고려해 읽음 완료는 optimistic UI로 먼저 반영한다.
서버 저장은 백그라운드로 처리하고, 실패하면 사용자에게 재시도 가능 상태를 보여준다.

성경 본문은 기존 로컬 데이터 기반 읽기 흐름을 유지하므로, 통독 화면 일부가 느려도 본문 읽기 자체는 가능한 구조를 지향한다.

## 단체 통독 확장 전략

단체 통독은 이번 MVP에서 제외한다.

다만 개인 통독이 실제로 사용되고, "함께 읽고 싶다"는 수요가 확인되면 후원 또는 유료 서버 전환을 검토한 뒤 다음 기능을 확장한다.

- 그룹 생성
- 초대 링크
- 멤버 참여
- 멤버별 장 완료 상태
- 그룹 진행률

이 결정은 무료 서버에서 무리하게 그룹 기능을 시작하기보다, 실제 수요와 운영 비용을 함께 검증하는 전략이다.

## 완료 기준

- 로그인 사용자가 통독 계획을 만들 수 있다.
- 사용자는 책 단위로 읽을 범위를 선택할 수 있다.
- 사용자는 하루 목표 장 수와 시작일을 정할 수 있다.
- 통독 이름은 자동 생성되며 사용자가 수정할 수 있다.
- 홈의 오늘의 말씀 아래에 나의 통독 카드가 표시된다.
- 앱은 오늘 읽을 말씀을 계산해 보여준다.
- 사용자는 기존 성경 읽기 화면에서 본문을 읽을 수 있다.
- 장 맨 아래에서 읽기 완료를 할 수 있다.
- 읽음 기록은 전체 진행률에 반영된다.
- 통독 중 하이라이트/저장/복사/공유는 기존 성경 읽기와 동일하게 동작한다.
- 진도표와 성경통독표에서 진행 상태를 확인할 수 있다.
- 완료한 장은 성경통독표에서 다시 열 수 있다.
- 사용자는 통독 계획을 삭제할 수 있다.
- 모든 장을 완료하면 새 통독 계획을 만들 수 있다.
- 완료 기록/히스토리/통계는 MVP에서 제외된다.
- 단체 통독은 향후 수요와 서버 비용 전환 후 확장할 기능으로 문서화된다.
