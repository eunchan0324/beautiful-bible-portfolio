# Home Plan

## 목적

홈 화면은 앱의 첫 진입점이지만, 현재 우선순위는 Spring Boot 백엔드 포트폴리오를 Phase 5까지 완주하는 것이다.
따라서 홈 화면은 지금 깊게 구현하지 않고, Phase 4/5의 핵심 백엔드 작업을 방해하지 않는 범위에서 역할만 정리한다.

## 현재 결정

- 홈 화면 고도화는 당장 구현하지 않는다.
- 오늘의 말씀은 홈 콘텐츠 전략이 확정될 때까지 보류한다.
- 이번주 말씀 이미지, 이미지 저장, 공유 기능은 좋은 아이디어지만 운영/UX 범위가 커지므로 Post-MVP로 미룬다.
- Phase 4에서는 홈 종속 기능보다 독립 기능인 성경 검색 API를 우선한다.

## 이유

오늘의 말씀과 이번주 말씀은 홈 화면의 메인 콘텐츠로 적합하지만, 다음 결정이 먼저 필요하다.

- 매일 바뀌는 오늘의 말씀으로 갈지
- 일주일 단위의 말씀 이미지 카드로 갈지
- 이미지 공유/저장을 핵심 기능으로 둘지
- 공동체 기능과 어떻게 연결할지

이 결정들은 제품/운영/디자인 성격이 강하다. 지금 단계에서 깊게 들어가면 검색, 배포, 예외 처리 같은 백엔드 포트폴리오 핵심 작업이 늦어진다.

## Phase 4 우선순위

1. 성경 검색 API 구현
   - PostgreSQL FTS 학습
   - 성경 앱 핵심 기능
   - 홈 화면과 독립적으로 동작 가능

2. 검색 프론트 연동 또는 간단 검색 화면
   - 책 이름 입력 시 본문 이동
   - 키워드 입력 시 관련 구절 목록 표시

3. 오늘의 말씀 재검토
   - 홈 화면 방향이 정해진 뒤 구현
   - 필요하면 `featured_verses` 또는 `weekly_featured_verses` 형태로 확장 가능하게 설계

## Post-MVP 아이디어

- 오늘의 말씀 카드
- 이번주 말씀 이미지 카드
- 이미지 저장/공유
- 주일예배 말씀 기반 큐레이션
- AI 기반 상황별 말씀 추천
- 공동체 묵상/나눔 기능

## 결론

홈 화면은 지금 구현하지 않고 방향만 정리한다.
다음 구현 작업은 Phase 4의 검색 API를 우선한다.
