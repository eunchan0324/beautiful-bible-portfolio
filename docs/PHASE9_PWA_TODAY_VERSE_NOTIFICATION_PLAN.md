# Phase 9: PWA 오늘의 말씀 알림

> PWA/Web Push를 사용해 매일 오늘의 말씀 알림을 보낸다.

---

## 배경

오늘의 말씀 알림은 사용자가 앱을 직접 열지 않아도 다시 만날 수 있는 기능이다.
무료 Render 서버의 cold start가 있어도, 알림 발송은 사용자가 화면에서 기다리는 작업이 아니므로 체감 부담이 비교적 작다.

---

## 목표

- 사용자가 알림을 허용하면 매일 오늘의 말씀 알림을 받을 수 있게 한다.
- Web Push 구독 정보를 백엔드에 저장한다.
- 외부 스케줄러가 백엔드 발송 API를 호출해 알림을 보낸다.

선행 문서:

- [오늘의 말씀 후보 선정 기준](PHASE9_TODAY_VERSE_CANDIDATES.md)

---

## 기술 구조

```text
PWA FE
  -> 알림 권한 요청
  -> Service Worker PushSubscription 생성
  -> BE에 구독 정보 저장

Scheduler
  -> 매일 정해진 시간 BE 발송 API 호출

BE
  -> 오늘의 말씀 선택
  -> 저장된 구독자에게 Web Push 발송
```

---

## 구현 범위

포함한다.

- 알림 구독 UI
- 브라우저 알림 권한 요청
- Push 구독 생성/해제
- Push 구독 저장 API
- 오늘의 말씀 알림 발송 API
- 외부 스케줄러 연동

포함하지 않는다.

- 사용자별 알림 시간 설정
- 교회 일정 알림
- 미완료 통독 리마인드
- 복잡한 알림 카테고리 관리

---

## 무료 서버 전략

Render 무료 서버는 잠들 수 있으므로 서버 내부 스케줄러만 믿지 않는다.

초기 후보:

- GitHub Actions schedule
- cron-job.org

발송 시간이 1~3분 늦어질 수 있는 것은 허용한다.

---

## 주의사항

- iOS는 홈 화면에 추가한 PWA에서만 Web Push가 기대 가능하다.
- 알림 권한은 사용자가 직접 허용해야 한다.
- 만료되거나 실패한 구독은 삭제해야 한다.
- 일반 사용자 API rate limit과 알림 발송 API는 분리해서 생각한다.

---

## 완료 기준

- 알림 구독/해제가 가능하다.
- 테스트 발송 API로 알림을 받을 수 있다.
- 외부 스케줄러에서 발송 API를 호출할 수 있다.
- 브라우저/iOS 제약을 사용자 안내에 반영한다.
