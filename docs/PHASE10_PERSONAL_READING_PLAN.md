# Phase 10: 개인 통독

> 개인이 통독 계획을 시작하고 진행률을 관리하는 기능.
> MVP 상세 기획은 [PHASE10_PERSONAL_READING_MVP_SPEC.md](PHASE10_PERSONAL_READING_MVP_SPEC.md)에서 관리한다.

---

## 배경

출석 체크 단독 기능은 보류했다.
대신 사용자의 실제 목적이 더 분명한 개인 통독을 만든다.

개인 통독은 단체 통독의 기반이 되며, 읽기 완료 데이터는 나중에 교회 캘린더나 그룹 진행률에도 확장할 수 있다.

---

## 목표

- 사용자가 개인 통독 계획을 시작한다.
- 오늘 읽을 본문을 확인한다.
- 읽음 완료를 기록한다.
- 전체 진행률을 확인한다.

---

## MVP 범위

포함한다.

- 개인 통독 계획 생성
- 통독 계획 목록/상세
- 오늘 읽을 본문 표시
- 읽음 완료 처리
- 진행률 표시
- 성경 읽기 화면과 연결

포함하지 않는다.

- 여러 종류의 복잡한 통독표
- 사용자별 세부 통계 대시보드
- 알림
- 단체 통독
- 출석 streak

---

## 데이터 초안

```text
reading_plans
- id
- user_id
- title
- start_date
- status
- created_at

reading_plan_items
- id
- plan_id
- day_number
- book_code
- chapter_num

reading_progress
- id
- plan_id
- user_id
- book_code
- chapter_num
- completed_at
```

---

## UX 방향

```text
통독
- 나의 통독 계획
- 오늘 읽을 본문
- 완료 버튼
- 전체 진행률
```

성경 읽기 화면에서는 해당 장이 통독 계획에 포함되어 있으면 완료 액션을 자연스럽게 보여준다.

---

## 무료 서버 전략

- 읽음 완료는 optimistic UI로 먼저 반영한다.
- 서버 저장은 백그라운드로 처리한다.
- 실패 시 다시 시도하거나 상태를 되돌린다.
- 진행률 조회는 캐시 가능하게 설계한다.

---

## 완료 기준

- 개인 통독 계획을 만들 수 있다.
- 오늘 읽을 본문을 볼 수 있다.
- 읽음 완료가 저장된다.
- 진행률이 계산된다.
- 단체 통독으로 확장 가능한 데이터 구조를 갖는다.
