# #68 마이페이지 사용자 화면 개선

## 배경

기존 마이페이지는 로그인 상태, 백엔드 계정, 저장 모드처럼 개발 중간 상태를 설명하는 문구가 많았다.
실서비스 화면에서는 사용자가 바로 이해할 수 있는 개인 기능 중심의 구조가 필요했다.

또한 #67에서 저장한 말씀 API가 준비되었기 때문에, 마이페이지에서 실제 사용자 데이터로 이어지는 진입점을 만들 수 있었다.

## 결정

- 마이페이지를 프로필 카드, 나의 말씀, 설정 진입점 중심으로 재구성했다.
- 프로필 수정은 별도 `/my/profile` 화면으로 분리했다.
  - 현재는 닉네임 수정만 제공한다.
  - 이후 프로필 이미지 변경 같은 기능을 붙일 수 있는 확장 지점으로 둔다.
- 읽기 설정과 로그아웃은 `/my/settings` 화면으로 분리했다.
- 저장한 말씀은 `/my/saved-verses` 화면으로 분리하고, #67 API와 연결했다.
- 최근 읽은 말씀은 아직 구현하지 않고 `곧 만나요` 상태로 노출했다.
- 하단 네비는 `/my/*` 하위 경로에서도 마이 탭이 활성화되도록 맞췄다.
- 홈 화면은 임시로 말씀 이미지를 풀스크린으로 보여주고, `성경 읽기` CTA만 제공하도록 변경했다.

## 저장한 말씀 UX

- 성경 읽기 화면의 구절 선택 액션에 저장 버튼을 추가했다.
- 여러 구절을 선택해 저장할 때 일부가 이미 저장되어 있어도 전체 실패처럼 보이지 않도록 처리했다.
  - 새로 저장된 구절이 있으면 성공 메시지를 보여준다.
  - 이미 저장된 구절은 그대로 둔다.
- 저장한 말씀 목록에서는 같은 책/장 안에서 연속된 구절을 하나의 묶음으로 보여준다.
- 묶음 삭제는 바로 실행하지 않고 확인 시트를 거치도록 했다.
- 삭제 아이콘은 휴지통 대신 체크된 북마크 아이콘을 사용해 “저장 해제” 느낌을 강화했다.

## 트레이드오프

홈 화면의 말씀 이미지는 아직 동적 데이터가 아니라 정적 이미지다.
다만 추후 “이번 주 말씀 이미지” 기능으로 확장할 때 홈의 역할과 레이아웃을 먼저 검증할 수 있다.

저장한 말씀 묶음 삭제는 사용성이 좋아지지만, 사용자가 개별 구절만 해제하고 싶을 때는 아직 세밀한 제어가 부족하다.
현재 단계에서는 단순한 목록 관리 경험을 우선했다.

## 검증

- `npx.cmd tsc --noEmit`

## 포트폴리오 포인트

- 개발 중간 상태의 설명형 마이페이지를 실제 사용자 기능 중심 화면으로 재설계했다.
- 백엔드 저장 API를 프론트 사용자 동선에 연결해 “읽기 → 저장 → 내 목록 확인 → 저장 해제” 흐름을 완성했다.
- 부분 성공, 중복 저장, 묶음 표시, 확인 시트처럼 실제 서비스에서 필요한 UX 세부 상태를 정리했다.
- 임시 홈 비주얼을 통해 이후 주간 말씀 이미지 기능으로 확장 가능한 UI 방향을 먼저 실험했다.
