# 100. 책 요약 데이터 구조와 화면 사용 방향

## 배경

장별 요약은 사용자가 특정 장을 읽기 전에 내용을 빠르게 파악하도록 돕는 기능이다. 책 요약은 이보다 한 단계 위에서, 사용자가 성경책 하나를 읽기 전에 책 전체의 흐름과 주제를 잡도록 돕는 안내 콘텐츠다.

책 요약은 66권만 생성하면 되므로 장별 요약보다 조금 더 풍성한 구조로 저장한다. 단순한 긴 문단 하나만 저장하면 상세 화면이 심심해지고 재사용도 어렵기 때문에, 화면 용도에 맞게 짧은 요약과 상세 구성 요소를 분리한다.

## 데이터 구조

`book_summaries`는 다음 필드를 가진다.

- `book_code`: 성경책 코드. 기존 `bible_books.book_code`와 같은 값이다.
- `short_summary`: 장 선택 화면 상단에 보여줄 1~2문장 요약이다.
- `summary`: 책 상세 페이지나 향후 AI 가이드 탭에서 보여줄 긴 설명이다.
- `reading_point`: 이 책을 읽을 때 붙잡을 핵심 관점 한 문장이다.
- `keywords`: 책의 핵심 키워드 목록이다. MVP에서는 JSON 문자열로 저장한다.
- `outline`: 책의 큰 흐름 목록이다. MVP에서는 JSON 문자열로 저장한다.
- `model`: 생성에 사용한 AI 모델 이름이다.

`keywords`와 `outline`을 별도 테이블이나 `jsonb`로 바로 분리하지 않는 이유는, 현재 목표가 검색/분석이 아니라 정적 콘텐츠 조회이기 때문이다. 우선 `TEXT`에 JSON 문자열로 저장하고, API 응답에서는 FE가 바로 쓰기 편하도록 `List<String>` 형태로 내려준다.

## 화면 사용 방향

### 장 선택 화면

장 선택 화면은 사용자가 장을 고르는 것이 핵심 목적이므로 긴 책 설명을 직접 펼치지 않는다. 대신 책 제목과 장 목록 사이에 `short_summary`를 작게 보여준다.

예시:

```text
레위기 - 총 27장

AI 책 요약
레위기는 예배와 정결, 거룩한 삶의 기준을 통해 하나님과 언약 백성의 관계를 보여 줍니다.
[자세히 보기]

[1] [2] [3] ...
```

`자세히 보기`를 누르면 바로 이동하지 않고 확인 모달을 띄워, 사용자가 장 선택 흐름에서 책 상세 설명 화면으로 이동한다는 것을 알 수 있게 한다.

### 책 상세 설명 화면

책 상세 설명 화면은 단순 텍스트 페이지가 아니라 다음 구성으로 읽히게 만든다.

- 책 이름
- `short_summary`
- `summary`
- `reading_point`
- `keywords`
- `outline`

이 구조는 나중에 하단 네비게이션의 임시 `AI 가이드` 탭에서도 재사용할 수 있다. 즉, 장 선택 화면은 가벼운 진입점이고, 상세 화면과 AI 가이드 탭은 더 자세히 읽는 공간이 된다.
