# #188 성경 읽기 TTS MVP

## 배경

Phase 14에서는 성경 읽기 화면에서 장 단위로 본문을 들을 수 있는 TTS MVP를 추가했다.
처음부터 외부 TTS API를 붙이면 비용, 음성 품질, 캐싱, 저작권, 서버 부하까지 함께 결정해야 하므로,
MVP에서는 브라우저 내장 Web Speech API(`speechSynthesis`)로 실제 사용성을 먼저 검증하는 방향을 선택했다.

현재 서비스는 Render 무료 서버를 사용하고 있고, 성경 본문은 프론트의 정적 JSON에서 읽는다.
따라서 TTS도 서버를 거치지 않고 브라우저에서 바로 실행하면 콜드스타트 영향을 받지 않고,
추가 API 비용 없이 듣기 경험을 빠르게 확인할 수 있다.

## 결정

- 성경 읽기 하단 컨트롤에 `듣기` 버튼을 추가했다.
- 듣기 중에는 기존 하단 컨트롤을 숨기고 전용 하단 플레이어를 표시했다.
- 플레이어는 재생/일시정지, 이전 절, 다음 절, 정지, 속도 선택을 제공한다.
- 속도 옵션은 체감이 약한 `1.2x` 대신 `0.8x / 1.0x / 1.5x`로 조정했다.
- 현재 읽는 구절은 하이라이트처럼 가로 전체 배경으로 표시했다.
- 듣기 중 구절을 누르면 선택/하이라이트 모드 대신 해당 절로 바로 점프한다.
- WEB 대조 역본이 켜져 있어도 TTS는 한국어 기본 본문 기준으로만 읽는다.

## 스크롤 읽기와 재생 대상 장

기존 성경 읽기 화면은 다음 장을 같은 페이지에 이어 붙이는 스크롤 구조다.
예를 들어 `/bible/룻/1` 페이지에서 아래로 내려 룻기 2장을 볼 수 있다.
이때 URL 기준 장은 여전히 1장이지만, 사용자가 보고 있는 장은 2장이다.

초기 구현에서는 URL의 장 번호나 로드된 첫 장을 기준으로 재생 대상을 잡아,
룻기 2장 화면에서 다시 듣기를 눌러도 룻기 1장이 재생되는 문제가 있었다.

이를 해결하기 위해 듣기 시작 시점에 실제 화면에서 가장 많이 보이는 장을 계산하고,
그 장을 TTS 대상 장으로 별도 상태에 저장했다.
또한 React 상태가 반영되기 전 이전 장의 verses를 읽지 않도록,
재생 명령을 바로 실행하지 않고 대상 장 상태가 렌더에 반영된 뒤 실행하게 했다.

## 트레이드오프

Web Speech API는 서버 비용이 없고 구현이 빠르지만, 브라우저와 OS 음성 엔진에 따라 품질과 속도 반영 정도가 달라질 수 있다.
특히 한국어 음성은 환경마다 억양과 속도 체감이 다르므로, MVP에서는 "정확히 같은 음질"보다 "성경 읽기 중 듣기 흐름이 유용한지"를 검증하는 데 집중했다.

외부 TTS API를 붙이면 더 안정적인 음성과 파일 캐싱이 가능하지만,
장별 오디오 생성/저장, 비용 관리, 재생 위치 동기화, 모바일 백그라운드 재생 정책까지 함께 설계해야 한다.
따라서 이번 단계에서는 브라우저 TTS로 기능 수요와 UX를 먼저 확인하고,
실사용 피드백이 충분할 때 서버 기반 TTS나 음원 캐싱을 검토하는 편이 안전하다.

## 검증

- `npx.cmd eslint src/app/bible/[bookId]/[chapter]/page.tsx src/components/ChapterNavigation.tsx src/components/VerseReader.tsx src/hooks/use-speech-synthesis-reader.ts`
- `npx.cmd tsc --noEmit --incremental false`
- `npm run build`
- 룻기 1장에서 듣기 시작 확인
- 룻기 1장 아래로 스크롤해 룻기 2장에서 듣기 시작 시 2장 1절부터 재생되는지 확인
- 듣기 중 구절 클릭 시 해당 절로 점프하는지 확인
- 이전/다음 절 버튼, 일시정지, 정지, 속도 선택 UI 확인

## 포트폴리오 포인트

- 서버 비용 없이 브라우저 Web Speech API로 TTS MVP를 구현했다.
- 스크롤 기반 성경 읽기 구조에서 URL 기준 장과 실제 화면 기준 장이 달라지는 문제를 찾아 재생 대상 계산 방식을 개선했다.
- React 상태 반영 타이밍 때문에 이전 장 데이터가 재생되는 문제를 대상 장 상태와 지연 실행 명령으로 분리해 해결했다.
- 하이라이트, 구절 선택, 대조 역본, 통독 읽기 화면과 충돌하지 않도록 기존 읽기 UX 위에 듣기 모드를 얹었다.
