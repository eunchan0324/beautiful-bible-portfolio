# 108. 인물 해설 데이터 구조와 샘플 콘텐츠

## 배경

인물 해설은 사용자가 성경 본문을 읽을 때 주요 인물의 흐름을 더 쉽게 이해하도록 돕는 안내 콘텐츠다. 설교나 묵상을 강요하기보다, 본문 안에서 확인할 수 있는 이야기 흐름과 관련 구절을 차분하게 정리하는 것을 목표로 한다.

이번 작업은 BE 구현 전 단계의 AI/Docs 기획 작업이다. 먼저 한 명의 샘플을 만들어 데이터 필드, 문장 길이, 화면 적합성을 확인한 뒤 FE 정적 데이터와 BE API 설계로 이어간다.

## MVP 방향

MVP에서는 인물 해설을 정적 콘텐츠처럼 다룬다. AI로 초안을 만들 수는 있지만, 실제 서비스에는 사람이 검수한 내용만 반영한다. 실시간 AI 호출은 사용하지 않는다.

이미지는 앱에서 자동 생성하지 않는다. GPT에서 직접 이미지를 만들고, 대화방에서 함께 검수한 뒤 프론트엔드 `public/images/persons/` 아래에 파일로 넣는다. 따라서 MVP 데이터에는 `imageUrl`을 두지 않고 `personCode`로 이미지 경로를 계산한다.

예상 이미지 경로:

```text
/images/persons/{personCode}.webp
```

예를 들어 `personCode`가 `abraham`이면 다음 파일을 사용한다.

```text
bb-bible-frontend/public/images/persons/abraham.webp
```

## 데이터 필드 초안

- `personCode`: 인물 고유 코드. 이미지 파일명과 라우팅에 사용한다.
- `name`: 한국어 이름.
- `shortDescription`: 목록과 상세 상단에 보여줄 한 줄 설명.
- `description`: 인물의 전체 흐름을 4~6문장 정도로 설명한다.
- `storyFlow`: 주요 이야기 흐름. 3~5개 단계로 나눈다.
- `keyVerses`: 핵심 구절 목록. `verseKey`는 기존 규칙처럼 `창1:1` 형식을 그대로 사용한다.
- `relatedBooks`: 관련 성경 책 코드 또는 이름.
- `keywords`: 인물 이해에 도움이 되는 태그.
- `cautionNote`: 사용자 노출용이 아니라 검수자가 참고하는 메모다.

## 화면 사용 방향

목록 화면에서는 `name`, `shortDescription`, `keywords`를 중심으로 보여 준다. 이미지는 `personCode`로 경로를 계산하되, 파일이 없으면 기본 placeholder를 보여 준다.

상세 화면에서는 상단 이미지, 한 줄 설명, 짧은 본문 설명, 이야기 흐름, 핵심 구절, 관련 책, 키워드 순서로 배치한다. 긴 문단 하나로 몰아 쓰지 않고, 사용자가 스크롤하면서 필요한 정보를 빠르게 잡을 수 있게 나눈다.

## 이미지 검수 기준

- 성경 인물을 현대 영화 캐릭터처럼 과장하지 않는다.
- 특정 인종이나 현대적 외모로 단정하지 않는다.
- 고대 근동 배경과 복장을 참고하되, 역사 재현을 지나치게 확정적으로 표현하지 않는다.
- 예수님, 하나님, 성령의 직접 이미지는 MVP에서 다루지 않는다.
- 인물 초상이 부담스러운 경우 장소, 상징, 사건 중심 이미지로 대체한다.
