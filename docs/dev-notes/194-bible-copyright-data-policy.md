# #194 성경 본문 저작권 및 공개 데이터 정책

## 배경

Beautiful Bible은 원래 우리 교회에서 실제로 읽는 성경책과 맞는 성경 앱이 부족하다는 문제에서 시작했다.
그래서 교회 실사용 버전은 저작권 보호 대상 한국어 본문을 기준으로 성경 읽기, 검색, 하이라이트, 저장한 말씀, 개인 통독, TTS 기능을 구현했다.

다만 포트폴리오 공개 단계에서는 상황이 달라진다.
GitHub 저장소와 배포 URL을 공개하면 성경 본문 데이터까지 외부에 공개될 수 있고, 운영에 사용하는 한국어 본문은 저작권 보호 대상이므로 별도 관리가 필요하다.

## 확인한 사실

- 현재 프론트엔드 `public/bible.json`과 백엔드 `resources/data/bible.json`에는 성경 전체 본문이 포함되어 있다.
- UI와 문서에서는 기본 본문을 운영 한국어 본문으로 표시하고 있다.
- 공개 안내 기준상 운영에 사용하는 한국어 성경 본문은 저작권 보호 대상일 수 있다.
- 비상업적 사용이어도 성경 본문 사용은 저작권 사용 허가가 필요할 수 있다.
- 성경 애플리케이션 또는 PC/모바일 웹을 통한 성경 본문 서비스는 별도 저작권료/허가 대상이 될 수 있다.

## 결정

### 1. 교회 실서비스 버전

교회 실사용자는 운영 한국어 본문에 익숙하므로, 실서비스 버전은 당분간 해당 본문 기준 UX를 유지한다.
다만 본문 저작권 리스크를 고려해 저장소는 public으로 공개하지 않고 private로 관리한다.

운영 원칙:

- 실서비스 저장소는 private로 관리한다.
- 배포 URL은 교회 내부 사용 중심으로 공유한다.
- 앱스토어/마켓 등록 또는 대규모 홍보는 사용 허가 확인 전까지 진행하지 않는다.
- 후원, 유료 서버 전환, 단체 통독 확장처럼 서비스 규모가 커지는 시점에는 대한성서공회 사용 허가 조건을 다시 확인한다.

### 2. 포트폴리오 공개 버전

포트폴리오에서 중요한 것은 특정 본문 자체가 아니라, 성경 서비스에 필요한 구조와 구현이다.
따라서 public 포트폴리오 저장소에는 저작권 보호 대상 본문 JSON을 포함하지 않는다.

공개 원칙:

- 포트폴리오 public repo에는 저작권 보호 대상 본문 데이터를 포함하지 않는다.
- README에 성경 본문 데이터는 저작권 이슈로 제외했다는 점을 명시한다.
- 공개 데모가 필요하면 개역한글, WEB, 또는 샘플 데이터를 사용한다.
- 코드 구조, API 설계, 사용자 데이터 모델, 통독, TTS, 역본 비교 흐름은 포트폴리오 설명의 중심으로 둔다.

## 포트폴리오 README 문구 초안

```md
성경 본문 데이터는 저작권 이슈로 공개 저장소에 포함하지 않았습니다.
실서비스 버전은 교회 사용성을 위해 별도 private 저장소에서 관리하며,
포트폴리오 공개 버전은 public domain 역본 또는 샘플 데이터를 주입할 수 있는 구조로 정리할 예정입니다.
```

## 면접 설명 문구

```text
Beautiful Bible은 실제 교회 사용자를 대상으로 시작한 서비스라 운영 버전에서는 교회에서 사용하는 한국어 본문을 기준으로 UX를 설계했습니다.
다만 포트폴리오 공개 단계에서는 성경 본문 저작권 리스크를 고려해 본문 데이터를 repository에서 분리하고,
공개 가능한 역본이나 샘플 데이터로 데모할 수 있도록 운영 버전과 공개 버전을 분리하기로 결정했습니다.
```

## 이후 작업 후보

- 권리자 또는 사용 계약 조건 확인
- 포트폴리오용 clean public repo 생성
- 저작권 보호 대상 본문 JSON 제외 및 `.gitignore` 정책 정리
- 개역한글 또는 WEB 기반 데모 데이터 구성
- README에 데이터 주입 방식과 저작권 정책 설명 추가

## 결론

지금 단계에서는 저작권 사용료를 지불해 특정 본문 기반 public 서비스를 운영하기보다, 교회 실서비스와 포트폴리오 공개 범위를 분리하는 것이 현실적이다.

운영 버전은 private로 관리하고, 포트폴리오 버전은 저작권 보호 대상 본문을 제외한 clean repo로 정리한다.
이 결정은 취업 준비 속도를 유지하면서도 저작권 리스크를 줄이기 위한 절충안이다.
