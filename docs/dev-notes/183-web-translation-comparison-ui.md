# #183 WEB 역본 비교 UI

## 배경

Phase 13에서는 한국어 성경 본문을 읽으면서 WEB 영어 본문을 함께 볼 수 있는 대조 역본 기능을 추가했다.
현재 성경 읽기 화면은 백엔드 API가 아니라 `public/bible.json` 정적 파일을 읽는 구조다.
Render 무료 서버가 잠들어 있어도 성경 읽기는 바로 가능하다는 장점이 있으므로, WEB 대조 본문도 같은 방향으로 정적 JSON을 사용했다.

## 결정

- 기본 본문은 운영 한국어 본문으로 고정했다.
- WEB은 기본 역본이 아니라 대조 역본으로만 제공했다.
- `public/bible-webp.json`을 추가하고, 기존 `verse_key` 형식을 그대로 사용했다.
- 대조 역본 선택값은 서버 설정이 아니라 `localStorage`에 저장했다.
- 하단 읽기 컨트롤의 `화면 모드`와 `글씨 크기` 사이에 역본 버튼을 배치했다.
- 대조 역본 선택지는 `선택 안 함`과 `WEB`만 제공했다.
- 영어 본문은 각 절 아래 보조 본문으로 표시했다.
- 하이라이트, 저장, 복사, 공유 기준은 계속 한국어 절의 `verse_key`로 유지했다.

## 트레이드오프

기본 역본을 WEB로 바꾸는 옵션도 가능하지만 MVP에서는 제외했다.
기본 본문을 영어로 바꾸면 검색, 오늘의 말씀, 통독, 공유, 저장 문구의 기준까지 함께 정해야 한다.
현재 사용자는 한국 교회 교인이므로 한국어 본문을 중심으로 두고, WEB은 이해를 돕는 보조 정보로 제한하는 편이 안전했다.

대조 본문을 백엔드 API로 제공하지 않은 것도 의도적인 선택이다.
영어 성경 본문은 로그인 사용자 데이터가 아니라 공용 읽기 데이터다.
프론트 정적 파일로 제공하면 서버 콜드스타트와 무관하게 읽기 경험을 유지할 수 있고, DB 마이그레이션 없이 먼저 사용성을 검증할 수 있다.

WEB 원문에는 긴 직접 인용에서 절마다 여는 따옴표가 반복되는 구간이 있다.
표시용으로 제거하면 화면은 더 자연스러워질 수 있지만, 원문을 임의로 바꾸는 일이므로 MVP에서는 그대로 표시하기로 했다.

## 검증

- `npx.cmd eslint src\components\ChapterNavigation.tsx src\components\VerseReader.tsx src\hooks\use-bible-store.ts src\types\bible.ts "src\app\bible\[bookId]\[chapter]\page.tsx"`
- `npx.cmd tsc --noEmit --incremental false`
- WEB 켜기/끄기 동작 확인
- 새로고침 후 WEB 선택 유지 확인
- 라이트/다크 모드 색상 확인
- 작은 글/큰 글 레이아웃 확인
- 레위기 2장의 WEB 원문 따옴표 표시 확인 후 원문 유지 결정

## 포트폴리오 포인트

- 서버 콜드스타트와 무관하게 동작하는 정적 성경 데이터 확장 구조를 유지했다.
- `verse_key`를 공통 식별자로 유지해 한국어 본문, WEB 본문, 하이라이트, 저장, 통독이 같은 절 위치를 공유하게 했다.
- 기본 역본 변경을 미루고 대조 역본만 제공해 MVP 범위를 작게 유지했다.
- 사용자 설정 저장을 서버가 아니라 localStorage로 시작해 실제 사용성을 먼저 검증하는 방향을 선택했다.
- 원문성 유지와 화면 자연스러움 사이에서 WEB 본문 표기를 임의 수정하지 않는 정책을 세웠다.
