# 아이디어 백로그

> 새 아이디어는 여기에 먼저 모은다.
> 우선순위가 정해지면 `PHASE_N_*.md` 기획서로 승격한다.
> 승격된 아이디어는 "→ Phase N으로 승격됨"으로 표시하고 이력으로 유지한다.

---

## 워크플로우

```
새 아이디어 발생
  ↓
이 문서에 항목 추가 (5~10줄, 가볍게)
  ↓
우선순위 결정 후 승격
  ↓
PHASE_N_PLAN.md 별도 작성 (상세 기획)
  ↓
이 문서에 "→ Phase N으로 승격됨" 표시
  ↓
구현 → dev-notes/N-*.md 회고
```

---

## 상태 범례

- 💡 **아이디어** — 떠올린 단계
- 🔍 **수요 확인 중** — 카톡방 설문 등으로 검증 중
- ⏸️ **보류** — 검증 결과 후순위 또는 수요 불확실
- ⏫ **승격됨** — Phase로 분리되어 별도 기획서 작성됨
- ✅ **완료** — 구현 완료

## 🤖 AI 활용

### AI 장/책/인물 요약 ⏫
사용자가 매번 보는 화면(성경 읽기)에 AI 사전 생성 콘텐츠 노출. 1,400개 사전 생성 + 캐싱 패턴.
→ **[Phase 7로 승격됨](PHASE7_AI_CONTENT_PLAN.md)**

### AI 의미 검색 (RAG) ⏸️
"위로가 필요할 때" 같은 자연어로 의미적으로 관련된 구절 검색. pgvector + 임베딩.
**보류 사유:** 실유저 수요 검증 부족. 카카오 로그인조차 잘 안 하는 패턴에서 AI 챗봇형 기능 수요 불확실.
**재검토 시점:** Phase 7/8 운영 데이터 누적 후.
→ 상세 기획: [PHASE7_RAG_PLAN_ARCHIVED.md](PHASE7_RAG_PLAN_ARCHIVED.md)

### 카톡 공유 + AI 이미지 카드 💡
구절 + AI 생성 배경 이미지를 카드 이미지로 만들어 카톡 공유. 어른들 카톡 공유 문화 + 외부 유입 채널.
**임팩트:** 어른 친화 ★★★★★ / 포트폴리오 ★★★ / 구현 부담 ★★★

### AI 묵상 가이드 💡
본문 읽고 묵상 질문 자동 생성 (예: "이 본문에서 하나님이 어떻게 보이나요?"). 사전 생성 + 캐싱 가능.
**임팩트:** 어른 친화 ★★★ / 포트폴리오 ★★★★★ / 구현 부담 ★★★★ (신학적 검수 필요)

## 📅 콘텐츠 운영

**임팩트:** 어른 친화 ★★★★★ / 포트폴리오 ★ / 구현 부담 ★★ (관리 화면 필요)

매일 자동 또는 수동 큐레이션한 말씀 노출. 결정론적 선택(`epoch % totalVerses`) 또는 본인이 직접 관리.
**임팩트:** 어른 친화 ★★★★ / 포트폴리오 ★★ / 구현 부담 ★

### 통독 챌린지 💡
1년 통독 진행률, 일자별 권장 본문 안내, 남은 일수 카운트. 출석 캘린더와 연결.
**임팩트:** 어른 친화 ★★★★ / 포트폴리오 ★★ / 구현 부담 ★★★

## 🔊 접근성

### 본문 낭독 (TTS) 💡
구절 옆 ▶ 버튼 → 외부 TTS API로 음성 재생. 시력 약한 분, 출퇴근/집안일 중 듣기.
**임팩트:** 어른 친화 ★★★★★ / 포트폴리오 ★★★ / 구현 부담 ★★

### 음성으로 말씀 찾기 (STT) 💡
"마태복음 5장 3절" 음성 입력 → 해당 페이지 이동. Web Speech API 무료 활용.
**임팩트:** 어른 친화 ★★★★★ / 포트폴리오 ★★ / 구현 부담 ★

### 글자 확대 / 시니어 UX 강화 💡
현재 글자 크기 설정 외에 고대비 모드, 행간/자간 조정 등.
**임팩트:** 어른 친화 ★★★★ / 포트폴리오 ★ / 구현 부담 ★★

## 평가 기준 가이드

새 아이디어 추가 시 다음 세 항목 ★1~5로 평가:

- **어른 친화** — 실제 교회 사용자(어른 비중 큼)에게 가치 있는가
- **포트폴리오** — 백엔드 포트폴리오로 임팩트 있는 기술 요소 있는가
- **구현 부담** — 1~2주 안에 1인 개발자가 완료 가능한가 (★ 적을수록 가벼움)

세 항목이 골고루 높은 아이디어 우선 승격.
