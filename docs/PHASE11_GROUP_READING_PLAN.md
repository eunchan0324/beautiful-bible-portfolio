# Phase 11: 단체 통독

> 목장/그룹 단위로 통독을 함께 진행하고 관리하는 기능.

---

## 배경

단체 통독은 포트폴리오와 실제 교회 사용성 모두에서 임팩트가 크다.
다만 그룹, 권한, 초대, 멤버별 진행률이 필요하므로 개인 통독 이후에 확장한다.

---

## 목표

- 사용자가 통독 그룹을 만든다.
- 초대 링크로 멤버가 참여한다.
- 그룹 통독 계획을 함께 진행한다.
- 관리자 또는 멤버가 전체 진행 상황을 확인한다.

---

## MVP 범위

포함한다.

- 그룹 생성
- 그룹 관리자/멤버 권한
- 초대 코드 또는 초대 링크
- 멤버 목록
- 그룹 통독 계획
- 멤버별 완료 여부/진행률 조회

포함하지 않는다.

- 복잡한 관리자 대시보드
- 미완료자 자동 알림
- 댓글/채팅
- 교회 전체 조직도
- 세밀한 통계 리포트

---

## 데이터 초안

```text
reading_groups
- id
- name
- owner_user_id
- created_at

reading_group_members
- id
- group_id
- user_id
- role
- joined_at

reading_group_invites
- id
- group_id
- invite_code
- expires_at
- created_by

group_reading_plans
- id
- group_id
- title
- start_date
- status
```

개인 통독의 `reading_progress` 구조를 재사용하거나 그룹 계획 ID를 연결하는 방식으로 확장한다.

---

## 권한 원칙

- 그룹 관리자는 초대 링크 생성, 멤버 목록 확인, 그룹 계획 생성을 할 수 있다.
- 멤버는 자신의 읽음 완료를 기록하고 그룹 진행률을 볼 수 있다.
- 모든 조회/수정은 `group_id`와 멤버십 검증을 통과해야 한다.

---

## 무료 서버 전략

- 멤버별 진행률은 한 번에 필요한 데이터만 조회한다.
- 큰 통계는 MVP에서 제외한다.
- 진행률 집계가 느려지면 별도 집계 테이블을 나중에 검토한다.

---

## 완료 기준

- 그룹을 만들고 초대 링크로 참여할 수 있다.
- 그룹 통독 계획을 만들 수 있다.
- 멤버별 완료 상태를 볼 수 있다.
- 권한 없는 사용자는 그룹 데이터에 접근할 수 없다.
